# Resume Bullets — Meridian: Real-Time Fraud Detection Platform

Use 3–5 bullets depending on role. All claims are scoped to synthetic-data portfolio work.

## FinTech-focused
- Built **Meridian**, a real-time fraud-risk scoring platform with analyst review queue workflows, decision traceability, and role-based access controls.
- Implemented fraud scenario simulation for deterministic demo runs and repeatable workflow validation across synthetic transaction cohorts.

## ML Engineer-focused
- Developed a hybrid fraud-scoring pipeline combining rule signals with logistic regression probability outputs and decision-band thresholds.
- Added SHAP-based transaction explainability to expose top feature contributions and improve analyst interpretation of model outcomes.

## Backend Engineer-focused
- Shipped FastAPI services for transaction ingest, scoring, explainability, review operations, audit logs, and job orchestration with JWT-authenticated APIs.
- Built test-backed API functionality and containerized local deployment workflows for reproducible full-stack demonstrations.

## Data Scientist-focused
- Designed synthetic fraud datasets and evaluated model behavior using precision, recall, F1, ROC-AUC, Brier score, and confusion-matrix analysis.
- Framed model results with explicit limitations and responsible-use caveats to avoid overclaiming production readiness.

## Safe interview phrasing
- “Portfolio project using synthetic/sample data; not connected to real banking systems.”
- “Evaluation results are illustrative and intended for comparative experimentation.”
- “Demonstrates architecture and workflow design for fraud operations, not a production fraud-prevention claim.”
